# AngelScript でホットリロードするサンプル

AngelScript の標準アドオン CSerializer を利用して、スクリプトのホットリロードを実装するサンプルです。

- モジュールを破棄するのではなく、アプリケーション全体の状態を保持したままホットリロードが行われます。

- [./AngelScript-HotReload/my_script/my_script.as](./AngelScript-HotReload/my_script/my_script.as) を変更すると、自動でホットリロードされます。

- グローバル変数や C++ 側で保持しているオブジェクトを一旦メモリ上へシリアライズを行い、スクリプトの再コンパイル後にデシリアライズして状態を復元することで実現するようです。

- `@` でオブジェクトを参照している場合でも問題なくシリアライズ / デシリアライズが出来るようです。参照先のオブジェクトは再コンパイル後に再構築されます。このとき、同じオブジェクトを参照している場合の再構築は元通りに復元されます。つまり、ObjA-->ObjC, ObjB-->ObjC のような再構築は、ObjA'-->ObjC1, ObjB'-->ObjC2 となるのではなく、ObjA'-->ObjC', ObjB'-->ObjC' と元通りの参照関係で復元されます。CSerializer の内部実装では、アプリケーション全体で参照関係を走査することで実現しているようです。

- 関数の中身の変更はもちろんのこと、クラスにメンバを追加したりしてもホットリロード後に元のアプリケーションの状態を復元が可能です。
